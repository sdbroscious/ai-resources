# spec-driven development pros and cons

## sdd pros
- shared context across humans and AI
- increased determinism (verifying AI followed instructions)
- better code (from richer context)
- architecture and compliance alignment
- improved decomposition of work (manage abundance)
- better documentation and traceability
- can facilitate larger scale work
- can facilitate greenfield work

## sdd cons
- waterfall trap
- doc maintenance burden
- non-determinism may not be eliminated
- can hinder smaller scale work
- can hard to apply to brownfield work
